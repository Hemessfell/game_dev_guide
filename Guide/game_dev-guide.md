# 📱 GUIA DE APRESENTAÇÃO: DESENVOLVIMENTO DE JOGOS

## 📖 EXPLICAÇÃO DA ÁREA

**Definição:**  
O desenvolvimento de jogos é um processo multidisciplinar que envolve programação, design, arte e áudio, utilizando motores gráficos como Unity ou Unreal Engine.

**Termos-Chave:**  
Unity, Unreal, Game Maker, Godot, RPG Maker, C#, C++, GML, HLSL, GDScript

---

## 💼 PROFISSÕES E ATUAÇÃO

### Responsabilidades

Apenas um adendo de que, apesar do desenvolvimento de jogos abranger diversas áreas de conhecimento, nesse documento focarei nas áreas de atuação relacionadas a **programação**, pois esse é o intuito do grupo de estudos.

As responsabilidades de um programador de jogos estão divididas em vários setores que podem ser abordados de forma especializada ou generalista. Alguns desses setores são:

- Construção e refinamento de **interfaces (UI)** como HUDs, menus e sistemas de inventário
- Desenvolvimento de **gameplay** como controles de personagens jogáveis e comportamentos de inimigos
- Implementação de **sistemas core** como sistema de save/load, sistema de economia, sistema de pause/resume e qualquer  outro sistema que "amarre" o jogo
- Desenvolvimento de **física customizável** para jogos que exigem uma simulação de física mais robusta do que a presente nas game engines
- Desenvolvimento de **shaders** utilizando a linguagem HLSL ou por meio de programação visual disponibilizada pelas próprias engines
- Programação e manutenção de **servidores online** e desenvolvimento de **multiplayer**
- Fazer a **portabilidade** de jogos para outras plataformas (consoles ou PC)

Vale lembrar que um programador de jogos pode atuar profissionalmente de diversas formas, como:

- Sendo o funcionário de uma empresa
- Trabalhando como freelancer para vários projetos
- Sendo um desenvolvedor solo
- Criando o próprio estúdio de jogos

### Impacto no Projeto
A programação é a "cola" do desenvolvimento de jogos. Tudo o que é idealizado em um jogo, é implementado por um programador. O programador tem a função de trabalhar em conjunto com todas as outras áreas de desenvolvimento para se certificar de que a visão artistica do jogo esteja sendo alcançada.

## 🛠️ TECNOLOGIAS UTILIZADAS

### Unity (C#)
A Unity é a game engine mais popular dentre as opções. Isso se deve pela extensiva cobertura de material de aprendizado na internet. Fóruns, documentação oficial, tutoriais, livros; tudo isso existe em grande quantidade, o que acaba facilitando muito o trabalho de quem está começando.

Outro ponto positivo da Unity é a versatilidade. É possivel fazer qualquer tipo de jogo nessa engine com relativa facilidade, desde jogos 3D com gráficos realistas a jogos 2D em pixel art.

A linguagem de programação utilizada pela Unity é o C#.

### Unreal (C++)
A Unreal é a segunda game engine mais popular e a engine padrão da indústria. Sabe aquele jogo AAA hiper-realista de empresas gigantes? Então, muito provavelmente esse jogo foi feito na Unreal, e se não foi, foi feito em uma engine própria baseada na Unreal.

Além disso, a Unreal tem o motor gráfico mais potente dentre as opções e é perfeita para fazer jogos 3D realistas, é realmente assustador o quão rápido é prototipar esse tipo do jogo na Unreal.

A linguagem utilizada na Unreal é o C++.

### Godot (GDScript)
A Godot é uma aposta extremamente promissora. Ela não é a engine mais completa, não chega perto de ter a quantidade de funcionalidades da Unity ou Unreal, mas tem uma vantagem que é praticamente inédita para engines desse porte: ela é open source, ou seja, você pode adicionar e modificar funcionalidades á vontade, além de ter acesso ilimitado ao código fonte.

A Godot está recebendo investimentos de vários estúdios de jogos e progressivamente melhorando a cada ano, além de ser incrivelmente leve, possibilitando que computadores menos potentes possam facilmente rodá-la.

Na Godot, o desenvolvedor também é capaz de fazer qualquer tipo de jogo (nesse sentido ela tem uma proposta parecida com a Unity), e está se tornando uma ótima porta de entrada para novos desenvolvedores.

A linguagem oficial da Godot é o GDScript, mas também é possível programar em C# nela.

### Game Maker Studio 2 (GML)
O Game Maker é de longe a engine menos versátil dentre as opções, sendo seu foco jogos 2D. 

A falta de versatilidade não significa jogos ruins, muito pelo contrário, como o foco da engine desde o começo foi o 2D, ela é capaz de fazer excelentes jogos desse tipo.

Alguns jogos feitos no Game Maker são: Hotline Miami, Katana Zero, Undertale, Hyper Light Drifter, Spelunky, entre outros.

A linguagem utilizada pelo Game Maker é o GML.
